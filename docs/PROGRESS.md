- На 2026-03-19 финальная de-legacy проверка в fresh runtime подтвердила: `OPENAI_MODEL_CLASSIFY` и `OPENAI_MODEL_ANALYZE` больше не влияют на routed model selection при явном `AI_LLM1/AI_LLM2_PROVIDERS_JSON`, но `OPENAI_API_KEY` пока нельзя удалять полностью из env из-за bootstrap-required поля в `Settings`; дополнительно подтверждено, что stale `AI_*_FIXED_ACCOUNT_ALIAS` остаются operational tail и должны чиститься вместе с pool config
- На 2026-03-19 formal closure Manual Output Validation отделил closed scope от non-blocking tails: full manager-card implementation, current worker issue, future `LLM-1` first-pass activation и возможная automation readiness preparation остаются отдельными next-phase вопросами и не блокируют завершение этой фазы
- На 2026-03-26 следующим согласованным шагом после закрытия Manual Output Validation зафиксирован `Manual Reporting Pilot`, а не automation readiness: ручной параметрический запуск отчётов, reuse existing artifacts, email-only delivery и bounded model/report-composer experiments теперь считаются ближайшим operating direction проекта
- На 2026-03-27 persisted-only assumption для operator reporting path больше не актуальна: current bounded slice уже source-aware, но `rop_weekly` CRM block по-прежнему остаётся placeholder, а report-composer пока оставлен как metadata slot + deterministic fallback
- На 2026-03-27 source-aware assumption теперь применима только к `manager_daily`; для `rop_weekly` standing rule уже другой: weekly path persisted-only by design
- На 2026-03-27 minimal weekly recipient fallback через live Bitrix `UF_HEAD` уже включён как standing runtime path, но richer org-structure logic beyond department-head resolution по-прежнему остаётся out of scope для текущего bounded slice
