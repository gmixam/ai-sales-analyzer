# CONCEPT_MVP1

## Статус документа
- Версия: 2.0
- Статус: актуально
- Этап: MVP-1 — Анализ звонков
- Горизонт MVP-1: 8–10 недель до GO/NO-GO

## Цель MVP-1
Построить систему, которая автоматически обрабатывает звонки отдела продаж по цепочке:
аудио → транскрипт → AI-анализ → отчёт менеджеру и руководителю.

MVP-1 должен дать первый измеримый результат:
- карточка каждого звонка;
- ежедневные ТОП-3 проблемные зоны;
- еженедельный coaching pack;
- динамика прогресса менеджеров;
- сводный отчёт по отделу;
- дашборд с историей оценок.

## Проблема, которую решает продукт
1. Менеджеры повторяют одни и те же ошибки, а обратная связь нерегулярна.
2. Нет объективного и измеримого контроля качества звонков.
3. Ценные сигналы от клиентов теряются и не доходят до продукта и руководителя.

## Границы MVP-1
### Входит в MVP-1
- Получение звонков из OnlinePBX
- Фильтрация звонков и привязка к менеджеру
- Транскрипция звонков
- LLM-классификация и глубокий анализ
- Карточка звонка
- Daily / weekly / monthly reporting loop
- Трекинг прогресса менеджера
- Доставка отчётов через Email / Telegram
- Сохранение результатов в PostgreSQL
- Looker Studio dashboard

### Не входит в MVP-1
- Автоматическое создание задач в Bitrix24 по договорённостям
- Полный модуль контроля исполнения договорённостей
- Полный модуль «Голос клиента» как отдельный продуктовый слой
- Омниканальность: WhatsApp, Telegram-чаты, email, Zoom
- Другие отделы кроме пилотного
- Коммерциализация / multi-tenant слой

## Ключевые выходы MVP-1
### Для менеджера
- Карточка каждого звонка с оценкой и рекомендациями
- Ежедневный отчёт с ТОП-3 проблемными зонами
- Еженедельный coaching pack
- Динамика прогресса по критериям

### Для РОПа
- Coaching pack по менеджерам
- Сводный отчёт по отделу
- Системные паттерны ошибок и динамики
- Dashboard в Looker Studio

## Верхнеуровневый pipeline
1. Получение звонков из OnlinePBX API
2. Фильтрация: отсечение коротких, внутренних и нерелевантных звонков; привязка к менеджеру
3. Извлечение аудио из архива
4. Транскрипция MP3 → текст
5. LLM-1: классификация и быстрый структурный проход
6. LLM-2: глубокий анализ по чек-листу и контракту
7. Сохранение в PostgreSQL
8. Доставка карточек и отчётов

## Роль источников и интеграций
### OnlinePBX
Основной источник звонков для MVP-1.
Используется для:
- получения списка звонков;
- загрузки артефактов звонка;
- формирования interaction pipeline.

### Bitrix24 (read-only в MVP-1)
Используется на раннем этапе MVP-1 для:
- определения, каких менеджеров включать в пилот;
- сопоставления звонков с менеджерами;
- фильтрации и управленческого отбора звонков.

Важно: Bitrix24 в MVP-1 используется только как read-only источник для маппинга и отбора.
Автосоздание задач в Bitrix24 относится к MVP-2 «Договорённости».

Текущее уточнение для MVP-1 / Фаза 1:
до подключения Bitrix24 read-only и scheduler допускается временный manual pilot mode для первого live end-to-end прогона.
В этом режиме отбор звонков делается через явную test target config / whitelist, а доставка идёт только в тестовые каналы.
Для test-mode live validation STT временно может идти через Whisper по env-конфигурации, если это быстрее разблокирует первый живой прогон.

## Архитектурные принципы
- Python — основной язык реализации.
- Celery + Redis — очередь и асинхронные этапы.
- PostgreSQL — база данных для operational и analytical records.
- AssemblyAI — STT.
- OpenAI GPT-4o-mini / GPT-4o — классификация и глубокий анализ.
- Для STT / LLM-1 / LLM-2 допускается слой provider/account routing с отдельными pools для cost control, failover и A/B tests без изменения business contract анализа.
- FastAPI — API и управление.
- Docker Compose — основа запуска и изоляции.
- instruction_version обязателен на каждом анализе.
- Контракт анализа должен оставаться совместимым для BI.
- department_id должен быть заложен с первого дня.

## Артефакты качества анализа
### Checklist definition
Отдельный артефакт, который содержит:
- этапы оценки;
- критерии внутри этапов;
- шкалу;
- критические ошибки;
- правила интерпретации.

### Analysis contract
Отдельный артефакт, который содержит:
- результат применения checklist к конкретному звонку;
- score_by_stage;
- criteria_results;
- strengths / gaps / recommendations;
- agreements / follow_up;
- optional product_signals.

### Manager card
Человекочитаемый формат для менеджера и РОПа, который строится на основе analysis contract.

## Основные KPI MVP-1
### Критичные
- Охват менеджеров, получающих отчёты: не ниже 80%
- Скорость звонок → анализ → доставка: P90 ≤ 24 часа
- Доля провалов пайплайна: не выше 2–3% в день
- Качество AI на ручной проверке: не ниже 90%
- ТОП-3 проблемных зоны всегда формируются и читаемы

### Желательные
- Менеджеры открывают и читают отчёты: не ниже 70%
- Конверсия по этапам воронки на пилоте: минимум +2 п.п. к baseline
- Средняя оценка звонков растёт относительно старта пилота

## GO / NO-GO
### GO
Продолжаем, если:
- выполнены все критичные KPI;
- пилот стабилен 1–2 недели;
- менеджеры и РОП считают отчёты полезными;
- unit-cost подтверждён;
- есть эффект на конверсии или качестве работы.

### NO-GO
Останавливаемся, если:
- провалы > 2–3% в день держатся 3+ дня подряд;
- отчёты не используются спустя 2 недели пилота;
- стоимость анализа выше прогнозируемого эффекта.
